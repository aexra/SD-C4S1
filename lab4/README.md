## Задание 4

### Аттестация требований

#### 1. Проверка правильности требований

Выдвинутые требования в общем случае могут быть реализованы все без исключения. Плюсом к этому остается пространство для масштабирования проекта в любую сторону при выдвижении дополнительных требований в будущем.

#### 2. Проверка на непротиворечивость

Спецификация требований не содержит противоречий.

#### 3. Проверка на полноту

Система полна.

#### 4. Проверка на выполнимость

Все технические аспекты требований являются выполнимыми:
- UI/UX: Множество WEB-фреймворков и библиотек к ним
- SSO: Простота реализации подключения по OpenID к Keycloak или OpenLDAP на более низком уровне
